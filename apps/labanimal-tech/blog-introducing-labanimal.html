LabAnimal v0.2.0 发布：开源实验动物合规管理系统

我们很高兴地宣布 LabAnimal v0.2.0 正式发布！这是一个专为实验动物设施设计的合规管理系统，内置 AVMA 安乐死标准、动态笼位密度计算、21 CFR Part 11 电子签名、AAALAC 审计准备等核心功能。

为什么需要 LabAnimal？

实验动物设施的管理面临着独特的挑战：严格的合规要求、复杂的记录保存、频繁的审计检查。现有的解决方案要么价格昂贵（每年数万美元），要么功能不全，要么难以满足日益严格的监管要求。

LabAnimal 的目标是填补这一空白——提供一个开源、合规、易用的解决方案，让研究人员专注于科学研究，而不是繁琐的文书工作。

核心功能

1. 合规引擎

LabAnimal 的合规引擎是系统的核心，它将行业标准直接内置到系统中：

  • AVMA 安乐死验证：实时验证安乐死方法是否符合 AVMA Guidelines 2020，按物种、年龄、设备自动匹配合规方法
  • NRC 密度计算：基于 NRC Guide 2011 标准自动计算最大饲养密度，支持小鼠、大鼠、仓鼠、豚鼠、兔
  • 3R 原则验证：在协议审批时自动验证 Replace（替代）、Reduce（减少）、Refine（优化）原则
  • USDA 疼痛分类：自动标记 D 类（伴随疼痛）和 E 类（无法缓解的疼痛）实验

2. 21 CFR Part 11 电子签名

对于需要 FDA 合规的设施，LabAnimal 提供了完整的 21 CFR Part 11 电子签名支持：

  • 所见即所签：签署内容的 SHA-256 哈希
  • 审计追踪：完整的操作历史记录
  • 哈希链：每条审计记录包含前一条的哈希，防篡改
  • 报告签名：PDF 报告可嵌入 RSA 签名和 QR 码

3. 多租户 SaaS 架构

LabAnimal 采用 PostgreSQL 行级安全（RLS）实现严格的多租户数据隔离：

  • 18 个数据库表启用 RLS 策略
  • API 层强制 labId 隔离
  • 每个实验室只能访问自己的数据
  • 完整的审计日志记录所有操作

4. 完整的动物管理

LabAnimal 提供了实验动物全生命周期的管理功能：

  • 动物记录：物种、品系、基因型、来源、状态
  • 笼位管理：动态密度计算、颜色编码、超限锁定
  • 健康记录：体重、体温、疼痛评分（MGS/RGS）
  • 繁殖管理：配对、妊娠、断奶
  • 死亡报告：原因记录、尸检、AVMA 验证

开源与商业

LabAnimal 采用 Open Core 模式：

  • 开源核心（Apache-2.0）：动物管理、笼位、协议、健康记录、审计日志等核心功能
  • 商业增值：多租户 SaaS、高级报告、PayPal 订阅、优先支持

我们选择 Apache-2.0 而非 AGPL-3.0，是因为 Apache-2.0 对商业使用更友好，不会产生传染性，这有助于建立更广泛的社区和生态系统。

快速开始

开始使用 LabAnimal 非常简单： 

# 克隆仓库
git clone https://github.com/xxjzone01-cyber/labanimal.git

# 安装依赖
cd labanimal
pnpm install

# 配置数据库
cp .env.example .env
# 编辑 .env 填入数据库连接信息

# 运行数据库迁移
pnpm prisma migrate dev

# 启动开发服务器
pnpm dev

或者直接使用我们的 SaaS 版本：免费注册。
