LabAnimal v0.2.0 Released: Open-Source Lab Animal Compliance Management System

We are excited to announce the official release of LabAnimal v0.2.0! It is a compliance management system designed for laboratory animal facilities, featuring AVMA euthanasia standards, dynamic cage density calculation, 21 CFR Part 11 electronic signatures, and AAALAC audit preparation.

Why LabAnimal?

Laboratory animal facilities face unique challenges: strict compliance requirements, complex record-keeping, and frequent audits. Existing solutions are either expensive (tens of thousands of dollars per year), incomplete in functionality, or unable to meet increasingly stringent regulatory requirements.

LabAnimal aims to fill this gap — providing an open-source, compliant, and easy-to-use solution so researchers can focus on science rather than paperwork.

Core Features

1. Compliance Engine

The LabAnimal compliance engine is the heart of the system, embedding industry standards directly into the platform:

  • AVMA Euthanasia Validation: Real-time verification of euthanasia methods against AVMA Guidelines 2020, auto-matching compliant methods by species, age, and equipment
  • NRC Density Calculation: Automatic maximum stocking density calculation based on NRC Guide 2011, supporting mice, rats, hamsters, guinea pigs, and rabbits
  • 3Rs Principle Validation: Automatic verification of Replace, Reduce, and Refine principles during protocol approval
  • USDA Pain Classification: Automatic flagging of Category D (accompanied by pain) and Category E (unrelieved pain) experiments

2. 21 CFR Part 11 Electronic Signatures

For facilities requiring FDA compliance, LabAnimal provides complete 21 CFR Part 11 electronic signature support:

  • WYSIWYS: SHA-256 hash of signed content
  • Audit trail: complete operation history
  • Hash chain: each audit record includes the hash of the previous one, tamper-proof
  • Report signing: PDF reports can embed RSA signatures and QR codes

3. Multi-Tenant SaaS Architecture

LabAnimal uses PostgreSQL Row-Level Security (RLS) for strict multi-tenant data isolation:

  • 18 database tables with RLS policies enabled
  • API-layer enforced labId isolation
  • Each laboratory can only access its own data
  • Complete audit logging of all operations

4. Complete Animal Management

LabAnimal provides full lifecycle management for laboratory animals:

  • Animal records: species, strain, genotype, source, status
  • Cage management: dynamic density calculation, color coding, over-limit locking
  • Health records: weight, temperature, pain scoring (MGS/RGS)
  • Breeding management: pairing, gestation, weaning
  • Death reporting: cause recording, necropsy, AVMA validation

Open Source & Commercial

LabAnimal uses an Open Core model:

  • Open-Source Core (Apache-2.0): Animal management, cages, protocols, health records, audit logs, and other core features
  • Commercial Add-ons: Multi-tenant SaaS, advanced reports, PayPal subscriptions, priority support

We chose Apache-2.0 over AGPL-3.0 because Apache-2.0 is more business-friendly and non-copyleft, which helps build a broader community and ecosystem.

Quick Start

Getting started with LabAnimal is easy: 

# Clone the repository
git clone https://github.com/xxjzone01-cyber/labanimal.git

# Install dependencies
cd labanimal
pnpm install

# Configure the database
cp .env.example .env
# Edit .env with your database connection info

# Run database migrations
pnpm prisma migrate dev

# Start the dev server
pnpm dev
