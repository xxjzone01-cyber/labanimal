🚀 快速开始

环境要求

  • Node.js ≥ 20
  • pnpm ≥ 9
  • PostgreSQL ≥ 15

1. 克隆仓库

# 克隆项目
git clone https://github.com/xxjzone01-cyber/labanimal.git
cd labanimal

# 安装依赖
pnpm install

2. 配置数据库

# 复制环境变量模板
cp packages/api/.env.example packages/api/.env

# 编辑 .env，设置 PostgreSQL 连接串
# DATABASE_URL="postgresql://user:password@localhost:5432/labanimal"

# 运行数据库迁移
pnpm --filter @labanimal/db push

# 生成 Prisma Client
pnpm --filter @labanimal/db generate

# 填充示例数据（可选）
pnpm --filter @labanimal/db seed

3. 启动开发服务器

# 同时启动 API + 前端（Turborepo）
pnpm dev

# API: http://localhost:3001
# 前端: http://localhost:5173

4. 运行测试

# 运行全部测试（103 个用例）
pnpm --filter @labanimal/api test

# 监听模式
pnpm --filter @labanimal/api test:watch

📡 API 参考

API 基础路径：/api，所有端点需要 JWT Bearer Token（除注册/登录外）。

认证

方法路径说明
POST/api/auth/send-code发送邮箱验证码
POST/api/auth/register注册（需要验证码）
POST/api/auth/login登录，返回 JWT

动物管理

方法路径说明
GET/api/animals?labId=xxx获取动物列表
POST/api/animals创建动物
PUT/api/animals/:id更新动物
DELETE/api/animals/:id删除动物

健康记录

方法路径说明
GET/api/health-records?labId=xxx获取健康记录
POST/api/health-records创建健康记录

IACUC 协议

方法路径说明
GET/api/protocols?labId=xxx获取协议列表
POST/api/protocols创建协议
PUT/api/protocols/:id更新协议

报告签名

方法路径说明
POST/api/license/signRSA 签名报告
POST/api/license/verify验证签名
GET/api/license/verify/:hash通过哈希公开验证

其他模块

模块路径前缀说明
房间/笼位/api/rooms, /api/racks, /api/cages设施管理
繁殖/api/breedings繁殖记录 + 断奶
死亡报告/api/death-reports死亡记录
药物/api/medications用药记录
审计日志/api/audit-logSHA-256 哈希链审计
电子签名/api/electronic-signatures21 CFR Part 11 签名
培训/api/trainings人员培训记录
API Keys/api/api-keys程序化访问密钥
订阅/api/subscriptionsStripe / PayPal 订阅

✅ 合规模块

AVMA 安乐死验证

内置 AVMA Guidelines 2020 规则引擎，按物种、年龄、设备自动匹配合规安乐死方法。

  • • 合规方法自动推荐
  • • 不合规方法自动阻断
  • • 见证人签名确认

动态笼位密度

基于 NRC Guide 2011 标准，实时计算最大饲养密度。

  • • 支持小鼠/大鼠/仓鼠/豚鼠/兔
  • • 超限自动锁定笼位
  • • 体重 + 笼位面积实时计算

21 CFR Part 11 电子签名

满足 FDA 电子记录和电子签名法规要求。

  • • SHA-256 哈希链审计追踪
  • • 签名含义声明
  • • 签名不可篡改

AAALAC 审计准备

一键生成 AAALAC 认证所需报告。

  • • 动物使用年度报告
  • • 死亡率统计
  • • 协议合规率

🏗️ 部署指南

选项 A：SaaS（推荐）

直接使用 labanimal.cloud，无需自行部署。

  1. 访问 labanimal.cloud/register
  2. 邮箱验证注册
  3. 选择套餐（Academic Free 永久免费）
  4. 创建实验室，开始使用

选项 B：自行部署

# 1. 克隆并安装
git clone https://github.com/xxjzone01-cyber/labanimal.git
cd labanimal && pnpm install

# 2. 构建
pnpm build

# 3. 配置环境变量
cp packages/api/.env.example packages/api/.env
# 编辑 .env 设置 DATABASE_URL、AUTH_SECRET 等

# 4. 运行数据库迁移
pnpm --filter @labanimal/db push
pnpm --filter @labanimal/db generate

# 5. 启动 API
cd packages/api && node dist/index.js

# 6. 部署前端（packages/app/dist）到任意静态托管

选项 C：私有部署（Enterprise）

适合需要完全数据控制的机构，$5,000 起（一次性付费）。

  • 离线授权码，无需联网验证
  • 30 天宽限期
  • 包含安装支持
  • 查看离线授权购买页 →

🔧 环境变量

变量必填说明
DATABASE_URL是PostgreSQL 连接串
AUTH_SECRET生产环境必填JWT 签名密钥（≥32 字符）
PORT否API 端口，默认 3001
CORS_ORIGIN生产环境必填允许的前端域名
BREVO_API_KEY是Brevo 邮件服务密钥
LICENSE_PRIVATE_KEY否RSA 私钥（报告签名用）
LICENSE_PUBLIC_KEY否RSA 公钥（签名验证用）
