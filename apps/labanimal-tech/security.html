安全白皮书

最后更新：2026 年 6 月 26 日

1. 安全概述

LabAnimal 将数据安全视为最高优先级。作为实验动物合规管理系统，我们处理的数据涉及研究协议、动物福利记录和审计追踪，这些数据的完整性和保密性至关重要。本文档详细说明了我们的安全架构和实践。

2. 数据加密

2.1 传输加密

  • 所有数据传输使用 TLS 1.3 加密
  • 强制 HTTPS，自动重定向 HTTP 请求
  • HSTS（HTTP Strict Transport Security）启用
  • 证书透明度（Certificate Transparency）日志

2.2 存储加密

  • 密码使用 bcrypt 哈希存储（cost factor 12）
  • JWT Token 使用 RS256（RSA + SHA-256）签名
  • 数据库连接使用 SSL
  • 敏感配置使用环境变量，不硬编码

3. 认证和授权

3.1 身份认证

  • JWT（JSON Web Token）无状态认证
  • 密码强度要求：8 位以上，包含大小写字母和数字
  • 登录失败速率限制：5 次/分钟
  • 注册速率限制：3 次/分钟

3.2 访问控制

  • 基于角色的访问控制（RBAC）：Owner / Admin / Member / Viewer
  • PostgreSQL 行级安全（RLS）：18 个表启用
  • API 层 labId 强制隔离：ensureLabId 中间件
  • 实验室成员权限验证

4. 多租户数据隔离

LabAnimal 采用严格的多租户隔离架构：

  • 数据库层：PostgreSQL RLS 策略确保行级隔离
  • API 层：每个请求自动注入 labId，无法跨租户访问
  • 应用层：前端组件按 labId 过滤数据
  • 审计层：所有操作记录在审计日志中，包含 labId

5. 审计追踪

为满足 21 CFR Part 11 合规要求，我们实现了完整的审计追踪：

  • AuditLog 模型：记录所有数据变更
  • SHA-256 哈希链：每条审计记录包含前一条的哈希，防篡改
  • 电子签名：支持所见即所签的电子签名
  • 报告签名：PDF 报告可嵌入 RSA 签名和 QR 码

6. 基础设施安全

6.1 托管环境

  • 前端部署在 Vercel（全球 CDN，自动 DDoS 防护）
  • API 服务器部署在安全的云服务器
  • PostgreSQL 数据库使用专用实例

6.2 网络安全

  • CORS 配置：多域名白名单
  • 请求速率限制
  • 输入验证（Zod schema）
  • SQL 注入防护（Prisma ORM 参数化查询）

7. 合规标准

LabAnimal 设计时考虑了以下合规标准：

  • 21 CFR Part 11：电子记录和电子签名
  • AVMA Guidelines 2020：安乐死方法验证
  • NRC Guide 2011：笼位密度计算
  • AAALAC：审计准备功能

8. 开源透明度

LabAnimal 的核心代码以 Apache-2.0 许可证开源，这意味着：

  • 任何人都可以审查我们的安全实现
  • 安全漏洞可以被社区发现和报告
  • 代码审计和验证完全透明
  • 没有隐藏的后门或恶意代码

9. 漏洞报告

我们重视安全漏洞的报告。如果您发现安全问题，请通过以下方式联系我们：

  • 安全邮箱：security@labanimal.tech
  • GitHub Security Advisories：提交漏洞

我们承诺在收到漏洞报告后 48 小时内响应，并在确认漏洞后尽快发布修复。

10. 安全更新

我们定期进行安全更新：

  • 依赖项安全审查（pnpm audit）
  • CI/CD 安全扫描
  • 硬编码密钥检测
  • 定期安全补丁更新
